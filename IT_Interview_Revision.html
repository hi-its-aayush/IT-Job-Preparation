This is what a typical day looks like. Interviewers ask this to see if you understand the operational reality of the role.

8:30 AM
Arrive & Review Ticket Queue

Check ServiceNow for overnight/early-morning tickets. Triage by priority: 2 P2s waiting (user can't log in, printer down), 5 P3/P4s (software requests, password resets, slow internet). Run through Slack for any urgent messages from management or helpdesk.

8:45 AM
Jump on P2: Account Lockout

User locked out on 3rd attempt. Verify identity via manager, unlock in dsa.msc, reset password, check if old password cached on phone (Teams/Outlook running old creds). Update ticket, close with user confirmation.

9:15 AM
New Staff Onboarding Handoff

IT coordinator hands over new hire. Verify AD account created, email synced, Intune enrolled device, Teams licenses assigned. Walk them through login, show them the help desk Slack channel, confirm access to shared drives. Document in ticket.

10:00 AM
Handle Printer Escalation

Printer on 2nd floor down. Not a hardware issue — checked toner, power. Network ping succeeds but print jobs stuck. Restart Print Spooler on the device (services.msc), clear queue, restart. Users back in 10 minutes. Document the fix in KB.

10:45 AM
Routine Helpdesk Support

Handle 3–4 quick wins: password resets, add user to distribution list, unlock shared drive access, install requested app via Intune. Each ~5 mins. Update status in ServiceNow, move to next.

11:30 AM
Remote Collaboration Issue

User on VPN, Teams call dropping. Test at teams.microsoft.com (web) — works fine. Desktop app issue. Clear cache (%appdata%\Microsoft\Teams), restart. Reconnect to VPN, test again. Resolved.

12:00 PM
Lunch + Standup

30-min standup with IT team. Report on morning tickets, escalations handled, any blockers. Manager asks about device enrollment status for new procurement batch. Note for PM.

1:00 PM
Knowledge Base / Documentation

Write up the printer Spooler fix for KB (so others can handle it). Update common password reset guide with new MFA reset procedures. 30 mins — small wins that scale support.

1:45 PM
Device Enrollment & Mobile Management

3 new hires' devices need Windows Autopilot setup. Configure profiles in Intune, trigger enrollment, monitor for BitLocker/compliance. One device fails compliance (old WiFi driver). Push driver update via WSUS, sync device. All enrolled by 2:30.

2:45 PM
Email/Exchange Issue

User can't see shared mailbox after being added. Check M365 Admin Center — license assigned, group added. Restart Outlook. Works. Mention to user they may need to restart their client for permissions to cache locally.

3:30 PM
Escalation & Networking Question

User on floor 3 slow internet. Isolate: just them. Ping 8.8.8.8 (good), ping google.com (fails) — DNS issue. ipconfig /flushdns, /release, /renew. Check with Meraki dashboard — AP on floor 3 is fine. Restart docking station + reboot. Fixed.

4:15 PM
Offboarding Request

Manager requests account disable for departing employee. Verify via HR, disable AD account (dsa.msc), revoke Entra ID sessions, remote wipe device (Intune), collect hardware checklist. Follow offboarding process to completion. Archive ticket.

5:00 PM
End-of-Day Wrap

Review ServiceNow: 18 tickets closed today, 3 escalated to L3 (one networking issue needs ISP involvement, one Exchange mail rule issue, one hardware RMA). Update status on open tickets. Prep tomorrow's list. Handoff notes for night shift.

🔑 Identity & Access (Active Directory / Entra ID)

Local Directory
Active Directory Users & Computers (dsa.msc)
Your daily workhorse. Manage on-prem users, computers, groups, and OUs.
Day-to-day:
  • Create/delete user accounts
  • Reset passwords + force change at next logon
  • Unlock locked-out accounts
  • Disable departing employees
  • Add/remove group memberships
  • Move users between OUs (applies different GPOs)
  • Check account properties (expiry, last login, etc.)
Cloud Identity
Entra ID (Azure AD) Admin Center
Cloud-based identity for hybrid/cloud-only organizations. Manages users synced from on-prem or cloud-native accounts.
Day-to-day:
  • View/manage cloud users (synced from AD via AADConnect)
  • Reset MFA for users locked out of authentication
  • Revoke all active sessions (emergency access revocation)
  • Assign Conditional Access policies (block risky sign-ins)
  • Check device compliance (enrolled in Intune?)
  • Manage password reset permissions (SSPR delegation)
Policies
Group Policy Management Console (gpmc.msc)
Push settings from one place to thousands of machines. L1/2 usually don't create GPOs, but you run gpupdate /force.
Day-to-day:
  • Run gpupdate /force when user applies policy immediately
  • Check which policies apply to a machine (gpresult)
  • Know common GPO purposes: password policy, desktop lockdown, mapped drives, software restriction
  • Can create basic GPO (password complexity) if trained
System Management
Services Manager (services.msc)
Start/stop/restart Windows services. L1/2 use it for printers, networking, updates.
Day-to-day:
  • Restart Print Spooler (stuck print jobs)
  • Stop/start Windows Update service
  • Restart BITS (Background Intelligent Transfer Service) for sync issues
  • Check if services are set to auto-start on reboot
  • Know which service = which function

🌐 Networking & Connectivity

Network Config
Command Prompt / PowerShell
Run ipconfig, ping, tracert, nslookup, netstat to troubleshoot network issues.
Day-to-day:
  • ipconfig /all — full network details (IP, gateway, DNS)
  • ipconfig /release + /renew — refresh DHCP lease
  • ipconfig /flushdns — clear local DNS cache
  • ping 8.8.8.8 -t — continuous ping (check packet loss)
  • ping google.com — DNS test (if fails but 8.8.8.8 works = DNS issue)
  • tracert google.com — trace hops (find where connection breaks)
  • nslookup example.com — check DNS resolution
  • netstat -an — view active connections (check for stuck services)
VPN
Cisco AnyConnect (or similar VPN client)
Remote access to corporate network. Common issues: won't connect, dropped sessions, slow speed.
Day-to-day:
  • Troubleshoot VPN connection failures
  • Verify user credentials haven't expired
  • Check VPN client version (outdated?)
  • Test from mobile hotspot (isolate local network issues)
  • Understand VPN split tunneling (route only corporate traffic through VPN)
  • Know when to escalate to network team (ISP blocking VPN port?)
Network Connections
Network Settings (ncpa.cpl)
Configure IP, DNS, adapter settings, wired/wireless troubleshooting.
Day-to-day:
  • Set static IP (if DHCP failing)
  • Change DNS servers (point to 8.8.8.8 for quick test)
  • Disable/enable network adapter
  • Check connection status (connected? speed?)
  • View IPv4/IPv6 settings
  • Know difference between Ethernet and WiFi settings

🔒 Security & Encryption

Encryption
BitLocker (Full Disk Encryption)
Encrypts entire disk. L1/2 doesn't manage it, but you retrieve recovery keys.
Day-to-day:
  • Get recovery key from Intune if user forgets PIN
  • Understand recovery key = master unlock (never share casually)
  • Confirm BitLocker enabled on managed devices (compliance check)

Windows Security
Windows Defender / Antivirus
Built-in AV. Managed via Intune policies. Users usually don't manage it.
Day-to-day:
  • Verify Defender is enabled & up-to-date (Settings → Virus & threat protection)
  • Quarantine suspicious files
  • Understand it's managed by group policy (IT controls updates)
  • Know when to escalate (malware suspected → security team)
Password Management
Self-Service Password Reset (SSPR)
Users reset their own password without calling helpdesk. Reduces tickets by 20%+.
Day-to-day:
  • Know SSPR portal (usually account.activedirectory.windowsazure.com)
  • Understand users must register phone/email beforehand
  • If user can't use SSPR → you reset manually in M365 Admin
  • Encourage adoption (reduces your workload)
MFA
Microsoft Authenticator / MFA
Two-factor authentication. Users register device in M365 Admin or Entra ID.
Day-to-day:
  • Reset MFA if user loses phone (M365 Admin → user → Manage MFA)
  • Understand MFA methods: app, SMS, phone call, hardware key
  • Know TOTP (time-based one-time password) if user uses authenticator app
  • Emergency: use backup codes if user locked out of all MFA methods

🌐 Internet slow / not working›
1
Isolate scope. One user or whole office? One = device. Whole = upstream.
2
Run ping 8.8.8.8 -t — look for packet loss in the output.
3
Run ping google.com — 8.8.8.8 works but this fails? DNS issue.
4
ipconfig /all — is the IP a 169.254.x.x APIPA address? DHCP failed.
5
Run ipconfig /release → /renew → /flushdns.
6
Restart docking station. Try a different cable.
7
WiFi: check Meraki — is the AP online? Corporate vs guest network?
Whole office affected → contact ISP and escalate to network team immediately.
🔒 Can't log in / password reset›
1
Verify identity first — confirm full name, employee ID, or manager before making changes.
2
Check if locked: dsa.msc → user → Properties → Account tab → Unlock account.
3
Reset password → right-click → Reset Password → tick "User must change at next logon".
4
Check Event Viewer — which machine is triggering the repeated lockout?
5
Ask: does the user have Outlook or Teams on their phone using the old password?
Most common cause: a phone with a saved old password hammering the account.
