Technical
What's the difference between a Distribution List and a Security Group?
›
"A Distribution List is email-only — you send one email and it fans out to all members. It can't be used for permissions. A Security Group controls access — who can get into a shared drive, SharePoint site, or application. Security groups can also be mail-enabled, so you get both functions in one. I create and manage both in M365 Admin Centre."
Technical
Walk me through what you'd do if a user says their internet is slow.
›
"First I ask: is it just them, or others on the same floor? That tells me if it's the device or the network. Then I run ping 8.8.8.8 -t to check for packet loss, and ping google.com — if 8.8.8.8 works but google.com doesn't, it's a DNS issue, not connectivity. Then ipconfig /release and renew to refresh the lease, flushdns to clear cache. I check their IP isn't an APIPA 169.254 address. If they're on WiFi I check the Meraki dashboard for AP status. If the whole office is affected, I contact the ISP and escalate to the network team."

This is the most common technical scenario question. Work through it methodically: isolate → diagnose → fix. See the full playbook in Troubleshooting.

🌐 Internet slow / not working ›
1
Isolate scope. One user or whole office? One = device. Whole = upstream.
2
Run ping 8.8.8.8 -t — check for packet loss in the output.
3
Run ping google.com — if 8.8.8.8 works but google.com fails → DNS issue.
4
Check IP with ipconfig /all. APIPA 169.254.x.x = DHCP failed.
5
Run ipconfig /release → ipconfig /renew → ipconfig /flushdns.
6
Restart docking station. Try different cable.
7
WiFi: check Meraki dashboard — is the AP online? Is the user on corporate vs guest WiFi?
Whole office affected → contact ISP + escalate to network team immediately.
📧 Outlook not syncing ›
1
Check status bar (bottom-right of Outlook). Should say "Connected to Exchange".
2
Test at office.com → OWA. If it works there, problem is the local client.
3
Launch Outlook Safe Mode: Ctrl + click the icon. If it works → add-in conflict.
4
Repair Office: Control Panel → Programs → Microsoft 365 → Repair (Quick first).
5
Rebuild OST: close Outlook → delete .ost file in %localappdata%\Microsoft\Outlook\ → reopen.
6
New profile: Control Panel → Mail → Show Profiles → Add new → set as default.
Always check M365 Service Health first — if Exchange is down, none of this matters.
🖨 Printer not working ›
1
Isolate: just this user, or everyone? Just this printer, or all printers?
2
Physical check: is printer on, no error lights, paper loaded, not showing offline?
3
Ping printer IP: ping [printer IP] — can't reach it? Network/IP issue.
4
Restart Print Spooler: services.msc → Print Spooler → Restart.
5
Remap printer: Win+R → \\server\printername
6
If stuck queue: stop Spooler → delete files in C:\Windows\System32\spool\PRINTERS → restart Spooler.
🔒 Account locked out ›
1
dsa.msc → find user → Properties → Account tab → tick "Unlock account".
2
Check Event Viewer for source of lockout — which machine triggered it?
3
Ask: do you have Outlook or Teams on your phone with old password saved? That can keep re-locking it.
4
Advise user to update saved credentials on all devices after password reset.
Most common cause: saved credentials on phone still using the old password.
🎧 Teams / Audio not working ›
1
Check Windows Sound Settings → correct output device selected (not laptop speakers).
2
Settings → Privacy → Microphone → toggle ON for Teams.
3
In Teams: ··· → Settings → Devices → select correct mic/speaker.
4
Clear Teams cache: %appdata%\Microsoft\Teams → delete contents → restart Teams.
5
Test on teams.microsoft.com (web) — isolates desktop client vs account.
🔑 VPN not connecting ›
1
Confirm user has working internet without VPN first.
2
Disconnect → reconnect. Check VPN client is up to date.
3
Check credentials — has their password expired?
4
Reinstall the VPN client.
5
Test from mobile hotspot — rules out local network blocking VPN ports.
6
Escalate with: error message, screenshots, steps tried.
💻 Laptop reported stolen ›
1
Confirm genuine theft with manager (not just misplaced).
2
Intune → Devices → Remote Wipe. Removes all company data immediately.
3
Entra ID → Revoke all sign-in sessions for the user.
4
Disable the user's AD account if credential compromise is suspected.
5
Log full incident in ticketing system with timestamps.
6
Report to building security / police. Begin replacement hardware process.
Never skip the Remote Wipe step — this is the most critical action and must happen immediately.
⚠️ "Trust relationship failed" ›

Machine account password out of sync with the Domain Controller.

1
Log in as local admin (not domain account).
2
PowerShell fix: Test-ComputerSecureChannel -Repair -Credential (Get-Credential)
3
Classic method: sysdm.cpl → remove from domain → rejoin domain → restart.
PowerShell method is faster — no full disjoin/rejoin needed on modern Windows.
